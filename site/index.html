Android-native local AI assistant host

AndroidClaw

A lightweight single-APK assistant runtime for Android that keeps chat, tools, skills, and scheduled automations on-device.

GitHub Repository Release GitHub Actions

Abstract

AndroidClaw is a Kotlin-first reinterpretation of the OpenClaw / NanoClaw runtime model for phones. Instead of transplanting a desktop Node.js host, it keeps the core contracts that matter on Android: durable chat sessions, typed tools, SKILL.md skills, and automation through WorkManager plus exact-alarm-aware scheduling.

Highlights

  • Visible streamed assistant output with cancel and retry.
  • Budgeted context assembly instead of a fixed last-N slice.
  • Typed tools and a persisted tool-call runtime loop.
  • Bundled, local, and workspace skills with slash dispatch.
  • Scheduler diagnostics, exact-alarm degradation, and task history.

Validation

  • Fast loop: :app:assembleDebug, unit tests, lint.
  • Release-like lane: :app:assembleQa, :app:assembleRelease.
  • Windows AVD smoke path from WSL.
  • Exact-alarm regression lane on API 34 / API 31 emulators.
